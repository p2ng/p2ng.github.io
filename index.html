    简介
    在2016年2月29日，入职阿里移动事业群-基础服务部-研发效能组，P5研发（达不到P6要求，非P6都签外包）
    团队规模
    共30+号人，外包占比（5/28），产品占比（4/28），P7+占比（4/28）
    工作环境
    上班时间
    09:00-09:30～18:00·18:30，弹性上下班。但一般不会准时拍拍屁股走人，在互联网公司不加班，你都会怀疑是不是进错公司了。
    待遇
    公司有专门饭堂（F14楼，红绿饭堂），包三餐（¥5，¥21，¥19的标准），自助餐形式（每月底有一次大餐，丰盛一些），吃饱喝足才有劲工作(加班)。
    薪酬(外包)：最多13薪，周末加班调休/结算日薪，日常没有所谓的加班补贴，社保公积金(¥190)买广州最低标准(甲方把一些非核心业务分离出来让外包支撑，就算月薪可能相当，但是公司运营层面上纳税会减少)
    下午茶
    各自团队handle，
    电脑
    可以自带电脑（以后不允许携带），或申请台式机（i5-3代、16G、SSD+HDD）、thinkpad（i5-3代、8G、HDD）、MacBook Pro/air（正式员工可以申请），可以申请双屏工作（搞网页开发，双屏还是有一点点效率提升的）。
    团队氛围
    晨会/主持人
    代码审查
    结对编程

    离职理由

  • 没有技术挑战，需要更大更刺激好玩的offer（个人）
  • 尽可能远离UI相关工作（个人）

术语

Application
基于Spark的用户程序，包含了driver程序和集群上的executor

Driver Program
运行main函数并且新建SparkContext的程序

Cluster Manager
在集群上获取资源的外部服务（例如standalone, Mesos, Yarn）

Worker Node
集群中任何可以运行应用代码的节点

Executor
是在一个worker node上为某应用启动的一个进程，该进程负责运行任务，并且负责将数据存在内存或者磁盘上。每个应用都有各自独立的executors

Task
被送到某个executor上的工作单元

Job
包含很多任务的并行计算，可以看做和Spark的action对应

Stage
一个Job会被拆分很多组任务，每组任务被称为Stage（就像MapReduce分map任务和reduce任务一样）

简介

简单说，词云是根据一段文字里的关键词出现频率生成文字的图片组合，字体越大说明对应的词汇的频率越高！词云适合分析一些文本类型的字段，比如用户职位分布、热点词分析等。
